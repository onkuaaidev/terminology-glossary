# Glossary

This file lists concise definitions. Use these as consistent vocabulary across notes and discussions.

- **state**: recorded facts about events (e.g., transaction recorded, timestamped).
- **treatment / behavior**: how an input was handled; the mapping from canonical input → outcome.
- **precedent**: institutional memory of how similar cases were treated previously.
- **policy version**: an explicit, versioned binding of rules that should be used to decide outcomes.
- **replayability**: the property that re-running identical canonical inputs against the same policy version produces the same outcome.
- **decision consistency**: guarantee that identical inputs yield identical decisions under the same policy/version.
- **executable commitment**: representation in software that both defines and enforces outcomes (not just records them).
- **anchor**: a human responsible for preserving consistency (institutional continuity).
